---
title: Properties
---

Properties hold the state of your component.

They are different from the `props` which are the inputs of your component.

```tsx
export class PostsList extends BaseComponent {
  title = 'Hello world'

  render() {
    return <h1>{this.title}</h1>
  }
}
```

<Callout type="warn">

Properties are different than your component `props`. Where `props` are the inputs and `properties` are your state.

</Callout>

## Initializing properties

Properties can be initialized when your component is mounted.

```tsx
export class PostsList extends BaseComponent {
  posts: Post[] = []

  async mount() {
    this.posts = await Post.findMany()
  }

  render() {
    return (
      <ul>
        {this.posts.map((post) => (
          <li>{post.title}</li>
        ))}
      </ul>
    )
  }
}
```

<Callout>

`mount` is a special method that is called only the first time your component is rendered.

</Callout>

## Data binding

Spark offers a `$model` prop to perform two-way data binding allowing you to synchornize data between a component and an input.

```tsx
export class PostsList extends BaseComponent {
  query = ''

  render() {
    const posts = await Post.query().whereLike('title', `%${this.query}%`)

    return (
      <div>
        <input type="text" $model={this.query} />
        <ul>
          {posts.map((post) => (
            <li>{post.title}</li>
          ))}
        </ul>
      </div>
    )
  }
}
```

In this example, when the user type inside the `input` it will ask the server to rerender the `PostsList` component with the new `query` value.
