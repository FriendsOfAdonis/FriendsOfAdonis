## `BaseAction`

The base class for all actions.

```ts
import { BaseAction } from '@foadonis/actions'

export default class MyAction extends BaseAction {
  async handle() {}
}
```

### Instance properties

| Property  | Type     | Description                                                          |
| --------- | -------- | -------------------------------------------------------------------- |
| `logger`  | `Logger` | Child logger scoped to the action's class name. Set on first invoke. |

### Static methods

| Method                | Description                                                                          |
| --------------------- | ------------------------------------------------------------------------------------ |
| `Action.run(...args)` | Resolves the action through the IoC container and invokes `handle(...args)`.         |

---

## `AsController()`

Mixin that makes an action runnable as an HTTP route handler. The action must implement `asController(ctx: HttpContext)`.

```ts
import { compose } from '@adonisjs/core/helpers'
import { BaseAction, AsController } from '@foadonis/actions'

export default class MyAction extends compose(BaseAction, AsController()) {
  async asController(ctx: HttpContext) {}
}
```

`AsController` takes no options.

---

## `AsCommand(options?)`

Mixin that registers the action as an Ace CLI command. Discovered automatically by [`ActionCommandsLoader`](#actioncommandsloader).

```ts
import { compose } from '@adonisjs/core/helpers'
import { BaseAction, AsCommand } from '@foadonis/actions'

export default class MyAction extends compose(
  BaseAction,
  AsCommand({ commandName: 'users:create' })
) {
  async asCommand(command: BaseCommand) {}
}
```

### Options

`AsCommandOptions` is `Partial<CommandMetaData>` from `@adonisjs/core/types/ace`. All fields are optional.

| Option        | Type             | Description                                                                                       |
| ------------- | ---------------- | ------------------------------------------------------------------------------------------------- |
| `commandName` | `string`         | The CLI invocation, e.g. `users:create`. Defaults to `action:<dash-cased-name>`.                  |
| `description` | `string`         | Shown in `node ace list`. Defaults to `Run the action <ActionName>`.                              |
| `aliases`     | `string[]`       | Alternate names for the command.                                                                  |
| `flags`       | `Flag[]`         | Boolean / string flags. See [Ace command metadata](https://docs.adonisjs.com/guides/ace/creating-commands). |
| `args`        | `Arg[]`          | Positional arguments.                                                                             |
| `options`     | `CommandOptions` | Command-level options like `staysAlive` and `startApp`.                                           |

## `AsListener<EventType>()`

Mixin that makes an action runnable as an event listener. The event type is inferred from the `asListener` parameter.

```ts
import { compose } from '@adonisjs/core/helpers'
import { BaseAction, AsListener } from '@foadonis/actions'

export default class MyAction extends compose(BaseAction, AsListener()) {
  async asListener(event: UserCreated) {}
}
```

`AsListener` takes no runtime options.

---

## `AsJob(options?)`

Mixin that registers the action as a queue job. Imported from the `@foadonis/actions/queue` subpath; requires `@adonisjs/queue` to be installed.

```ts
import { compose } from '@adonisjs/core/helpers'
import { BaseAction } from '@foadonis/actions'
import { AsJob } from '@foadonis/actions/queue'

interface Payload { userId: string }

export default class MyAction extends compose(BaseAction, AsJob<Payload>()) {
  async asJob({ userId }: Payload) {}
  async asJobFailed?(error: Error, payload: Payload, context: JobContext) {}
}
```

### Options

All [`@adonisjs/queue` `JobOptions`](https://docs.adonisjs.com/guides/digging-deeper/queues) are accepted. Common ones:

| Option     | Type     | Description                                                              |
| ---------- | -------- | ------------------------------------------------------------------------ |
| `name`     | `string` | Override the job name used by the worker. Defaults to the class name.    |
| `queue`    | `string` | Default target queue.                                                    |
| `priority` | `number` | Default priority (1–10, lower runs first).                               |
| `adapter`  | `string \| (() => Adapter)` | Default adapter.                                              |

### Static methods

Generated by the mixin on the action class:

| Method                                 | Description                                                                             |
| -------------------------------------- | --------------------------------------------------------------------------------------- |
| `Action.dispatch(payload)`             | Returns a [`JobDispatcher`](#jobdispatcher) for fluent configuration before dispatch.    |
| `Action.dispatchMany(payloads)`        | Returns a `JobBatchDispatcher` from `@adonisjs/queue` for batched dispatch.              |
| `Action.schedule(payload)`             | Returns a `ScheduleBuilder` from `@adonisjs/queue` for cron / interval scheduling.       |

### `JobDispatcher`

Returned by `Action.dispatch(payload)`. Awaiting the dispatcher dispatches the job (thenable); `.run()` is also available.

| Method            | Description                                                              |
| ----------------- | ------------------------------------------------------------------------ |
| `.toQueue(name)`  | Target queue (default: `'default'`).                                     |
| `.priority(n)`    | Priority 1–10 (lower runs first).                                        |
| `.in(delay)`      | Delay before processing (`'5s'`, `'1h'`, `'7d'`, or milliseconds).       |
| `.group(id)`      | Tag with a group id for monitoring tools.                                |
| `.with(adapter)`  | Use a specific adapter (string name or factory).                         |
| `.run()`          | Dispatch immediately. Returns `{ jobId }`.                               |

## `indexActions(options?)`

Assembler hook that generates a barrel file of lazy-loaded action loaders. Registered via `addAssemblerHook('init', 'indexActions()', true)` (added automatically by `node ace add @foadonis/actions`). See [Assembler hooks](https://docs.adonisjs.com/guides/concepts/assembler-hooks) for how the hook system works.

```ts title="adonisrc.ts"
import { indexActions } from '@foadonis/actions'

defineConfig({
  hooks: { onBuildStarting: [indexActions()] },
})
```

### Options

| Option          | Type       | Default              | Description                                                                                                |
| --------------- | ---------- | -------------------- | ---------------------------------------------------------------------------------------------------------- |
| `source`        | `string`   | `'app/actions'`      | Folder scanned for action files.                                                                           |
| `importAlias`   | `string`   | `'#actions'`         | Import alias used in the generated file.                                                                   |
| `glob`          | `string[]` | `['**/*_action.ts']` | Files included in the index.                                                                               |
| `skipSegments`  | `string[]` | `['actions']`        | Path segments to flatten when generating identifiers. With the default, `app/auth/actions/login_action.ts` resolves to `actions.Auth.Login`. |

The generated file lives at `.adonisjs/server/actions.ts` and is imported through `#generated/actions` (you must add the `#generated/*` alias yourself).

## `loader(import)`

Wraps a lazy import so it can be used as a route handler or event listener. Used internally by the generated barrel; rarely needed directly.

```ts
import { loader } from '@foadonis/actions'

const CreateUserLoader = loader(() => import('#actions/create_user_action'))

router.post('/users', CreateUserLoader.asController())
```

The `LoaderMethods` returned exposes `asController()` / `asListener()` only when the loaded action implements the matching mixin.

## `actions` service

```ts
import actions from '@foadonis/actions/services/main'
```

The default export is an [`ActionExecutor`](#actionexecutor) instance. Use it to register hooks. See [Hooks](/docs/actions/hooks).

---

## `ActionExecutor`

Resolves and runs actions. Exposed via the `actions` service and the `actions.executor` IoC binding.

| Method                       | Description                                                              |
| ---------------------------- | ------------------------------------------------------------------------ |
| `executor.hook(event, fn)`   | Register a hook. Currently supports the `'execute'` event.               |
| `executor.resolve(Action)`   | Resolve an action through the container with a hydrated logger.          |
| `executor.execute(action, method, handler, args)` | Internal dispatch path used by mixins.                       |

### `execute` hook

```ts
actions.hook('execute', (context) => {
  // optional cleanup
  return () => {}
})
```

| Property         | Type                                  | Description                                |
| ---------------- | ------------------------------------- | ------------------------------------------ |
| `context.action` | `BaseAction`                          | The resolved instance.                     |
| `context.method` | `'handle' \| 'asController' \| 'asCommand' \| 'asListener' \| 'asJob'` | The entrypoint name. |
| `context.handler`| `Function`                            | The original method being executed.        |
| `context.args`   | `unknown[]`                           | Arguments passed to the entrypoint.        |

## `ActionCommandsLoader`

Ace loader that exposes every action implementing `AsCommand` as a CLI command. Wire it in `bin/console.ts`:

```ts
import { ActionCommandsLoader } from '@foadonis/actions'

new Ignitor(APP_ROOT, { importer: IMPORTER })
  .ace()
  .configure(ActionCommandsLoader.configure)
  .handle(process.argv.splice(2))
```

The directory scanned is `app.rcFile.directories.actions ?? 'app/actions'`.

## `middlewares`

Pre-built middleware functions you can attach in `config/actions.ts`.

```ts
import { defineConfig, middlewares } from '@foadonis/actions'

export default defineConfig({
  middlewares: {
    asListener: [middlewares.asListener.scopedLogger],
    asController: [middlewares.asController.scopedLogger],
    asJob: [middlewares.asJob.scopedLogger],
  },
})
```

| Middleware                            | Effect                                                                              |
| ------------------------------------- | ----------------------------------------------------------------------------------- |
| `middlewares.asListener.scopedLogger` | Adds `action` and `event` fields to the logger.                                     |
| `middlewares.asController.scopedLogger` | Replaces the action logger with the `HttpContext` logger.                         |
| `middlewares.asJob.scopedLogger`      | Adds `action`, `queue`, `job_id`, `job_name` fields to the logger.                  |

## Subpath exports

| Subpath                            | Purpose                                                          | Required peers                                  |
| ---------------------------------- | ---------------------------------------------------------------- | ----------------------------------------------- |
| `@foadonis/actions`                | Core API: `BaseAction`, mixins, `loader`, `indexActions`, etc.   | `@adonisjs/core`                                |
| `@foadonis/actions/services/main`  | Default export of the `actions` service.                         | `@adonisjs/core`                                |
| `@foadonis/actions/commands`       | Ace commands (`make:action`).                                    | `@adonisjs/core`                                |
| `@foadonis/actions/actions_provider` | Service provider — registered by `node ace add`.               | `@adonisjs/core`                                |
| `@foadonis/actions/queue`          | `AsJob` mixin and dispatcher integration.                        | `@adonisjs/queue`, `@boringnode/queue`, `@lukeed/ms` |
| `@foadonis/actions/otel`           | `telemetryHook` for OpenTelemetry tracing.                       | `@opentelemetry/api`                            |
| `@foadonis/actions/girouette`      | `toGirouetteControllers(actions)` adapter for Girouette.         | `@adonisjs-community/girouette`                 |

## CLI commands

| Command                  | Description                          |
| ------------------------ | ------------------------------------ |
| `node ace make:action`   | Scaffold a new action under `app/actions/`. |
| `node ace action:<name>` | Run any action that uses `AsCommand` (auto-discovered). |
