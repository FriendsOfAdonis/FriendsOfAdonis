Beyond subscriptions, Shopkeeper lets you make one-off charges, create invoices for individual items, and issue refunds. This guide covers all the ways to charge customers outside of a subscription.

## Simple charge

To make a one-off charge against a customer's payment method:

```ts
router.post('/purchase', async ({ auth, request }) => {
  const user = auth.getUserOrFail()
  const paymentMethodId = request.input('paymentMethodId')

  const payment = await user.charge(100, paymentMethodId)
})
```

<Callout type="warn">
The `charge` method accepts amounts in the smallest currency unit. For example, $10.00 USD is `1000` (cents).
</Callout>

You can pass additional [Stripe charge options](https://stripe.com/docs/api/charges/create) as a third argument:

```ts
await user.charge(100, paymentMethodId, {
  description: 'One-time setup fee',
  metadata: { order_id: '12345' },
})
```

The `charge` method returns a Shopkeeper `Payment` instance on success, or throws an exception if the charge fails:

```ts
try {
  const payment = await user.charge(100, paymentMethodId)
} catch (error) {
  // Handle the failed charge
}
```

You can also charge without an existing customer by using a new model instance:

```ts
const payment = await new User().charge(100, paymentMethodId)
```

## Charge with invoice

If you need to generate an invoice for a one-off charge, use `invoicePrice`. This charges the customer's default payment method and creates a downloadable invoice:

```ts
// Invoice for 5 t-shirts
await user.invoicePrice('price_tshirt', 5)
```

You can pass additional options for both the invoice item and the invoice itself:

```ts
await user.invoicePrice(
  'price_tshirt',
  5,
  { discounts: [{ coupon: 'SUMMER21SALE' }] },
  { default_tax_rates: ['txr_id'] }
)
```

## Charge tabs

To build up a "tab" of multiple items before invoicing, use `tabPrice`:

```ts
await user.tabPrice('price_tshirt', 5)
await user.tabPrice('price_mug', 2)
await user.invoice()
```

For ad-hoc amounts without a pre-defined Stripe price, use `invoiceFor`:

```ts
await user.invoiceFor('One Time Fee', 500)
```

<Callout>
Using `invoicePrice` and `tabPrice` with pre-defined prices is recommended over `invoiceFor`, as it gives you better analytics in the Stripe dashboard.
</Callout>

<Callout type="warn">
The `invoice`, `invoicePrice`, and `invoiceFor` methods create Stripe invoices that automatically retry failed charges. If you don't want retries, close the invoice via the Stripe API after the first failed attempt.
</Callout>

## Payment intents

For more control over the payment flow, you can create a [payment intent](https://stripe.com/docs/payments/payment-intents) directly:

```ts
router.post('/pay', async ({ auth, request }) => {
  const user = auth.getUserOrFail()
  const payment = await user.pay(request.input('amount'))

  return payment.clientSecret()
})
```

The client secret is returned to your frontend, where the customer completes the payment using [Stripe.js](https://stripe.com/docs/payments/accept-a-payment?platform=web).

To restrict which payment methods are available:

```ts
const payment = await user.payWith(1000, ['card', 'bancontact'])

return payment.clientSecret()
```

<Callout type="warn">
The `pay` and `payWith` methods accept amounts in the smallest currency unit (e.g., cents for USD).
</Callout>

## Refunds

To refund a charge, use the `refund` method with the payment intent ID:

```ts
const payment = await user.charge(1000, paymentMethodId)

// Full refund
await user.refund(payment.id)
```
