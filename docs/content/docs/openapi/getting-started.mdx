## Installation

Install and configure the package using the following command :

```sh
node ace add @foadonis/openapi
```

## Getting started

### Decorate your controller

The OpenAPI schema is automatically generated using Typescript decorators. The routes of your application are automatically loaded but you might want to add some information like the description, response type, etc.

```ts twoslash title="app/controllers/posts_controller.ts"
// @noErrors
import Post from '#models/post'
import { createPostValidator } from '#validators/post'
import { ApiOperation, ApiBody, ApiResponse } from '@foadonis/openapi/decorators'

export default class PostsController {
  @ApiOperation({ summary: 'List all Posts' }) // [!code highlight]
  @ApiResponse({ type: [Post] }) // [!code highlight]
  index() {
    // ...your logic
  }

  @ApiOperation({ summary: 'Create a new Post' }) // [!code highlight]
  @ApiSchema(createPostValidator) // [!code highlight]
  @ApiResponse({ type: Post }) // [!code highlight]
  create() {}
}
```

The above decorators act as the following:

**`@ApiOperation()`**:  
Configures the operation. This is not required as the operations are automatically loaded from the Adonis Router.

**`@ApiResponse({ type: [Post] })`**:  
Defines the response return type to be a list of Posts.

**`@ApiSchema(createPostValidator)`**:  
Defines the request schema by using the Vine validator `createPostValidator`. [Read more](/docs/openapi/validators).

### Decorate your Model

It is now time to decorate your model to configure which properties are available.

```ts twoslash title="app/models/post.ts"
import { ApiProperty } from '@foadonis/openapi/decorators'

export default class Post {
  @ApiProperty() // [!code highlight]
  declare id: number

  @ApiProperty() // [!code highlight]
  declare title: string

  @ApiProperty() // [!code highlight]
  declare content: string
}
```

<Callout>
  Lucid decorators have been removed from the example for readability reasons but you can generate
  your OpenAPI schemas directly from your models.
</Callout>

### Create the Controller (optional)

<Callout>This step is done automatically when configuring the package.</Callout>

Create the controller that will serve your OpenAPI document:

```ts title="app/controllers/openapi_controller.ts"
export default class OpenAPIController extends BaseOpenAPIController {
  constructor() {
    super({ document: 'v1', ui: 'scalar' })
  }
}
```

Now register the routes for this controller inside `start/routes.ts` using the `openapi.registerController` method.

```ts title="start/routes.ts"
import openapi from '@foadonis/openapi/services/main'
import { controllers } from '#generated/controllers'

openapi.registerController('/api/v1', controllers.OpenApi).as('openapi')
```

### Access the documentation

You can now head over your OpenAPI documentation at [http://localhost:3333/api](http://localhost:3333/api). It will welcome you with a beautiful documentation page of your api thanks to [Scalar](https://scalar.com).

The following endpoints are available:

- [/api/v1](http://localhost:3333/api/v1) - Displays the documentation UI
- [/api/v1.json](http://localhost:3333/api/v1.json) - Returns the documentation in JSON format
- [/api/v1.yaml](http://localhost:3333/api/v1.yaml) - Returns the documentation in YAML format
